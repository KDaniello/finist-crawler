## Что это такое

Finist Crawler — инструмент для автоматического сбора текстовых данных с популярных платформ. Вы выбираете источник, вводите поисковый запрос и получаете структурированный файл Excel или CSV. Все в удобном интерфейсе — выбираете нужную платформу, сколько нужно спарсить и какие ключевые слова для сбора.

**Для кого:**
- Преподаватели, собирающие учебные банки данных для занятий
- Учёные-гуманитарии, собирающие корпусы текстов
- Маркетологи, анализирующие отзывы и обсуждения
- Журналисты, исследующие общественное мнение
- Аналитики, работающие с пользовательским контентом

## Как пользоваться

### 1. Выбор источника

На странице Запуск выберите платформу кликом по карточке.
Карточка подсветится зелёным.

### 2. Ввод параметра

В зависимости от источника введите:

- Ключевое слово — для Reddit, Хабра, Ленты, 2GIS
- App ID игры  — для Steam (число из URL страницы игры на store.steampowered.com)
- URL страницы — для Отзовика (https://otzovik.com/reviews/название/) | https://otzovik.com/reviews/kinopoisk_ru-onlayn-kinoteatr/ (Кинопоиск)

### 3. Настройка объёма

    Глубина поиска   Сколько страниц результатов просмотреть
    Объём сбора      Сколько материалов обработать (0 = все найденные)

### 4. Мониторинг

После нажатия «Начать парсинг» приложение автоматически переходит
на страницу мониторинга, где отображаются:

- Количество собранных записей в реальном времени
- Прогресс по каждой ветке парсинга
- Загрузка CPU и RAM
- Журнал событий

### 5. Экспорт результатов

На странице Результаты выберите сессию и нажмите CSV или XLSX.
Файл сохраняется рядом с данными в папке data/session_*/source_name/.

Замеры проведены на реальных данных (см. benchmark_results.json):

    Средняя скорость:         26.3 записей/сек
    Пик оперативной памяти:   238 MB
    Успешных источников:      5/5

Детализация по источникам:

    Источник    Время    Записей   Скорость     RAM пик
    ────────────────────────────────────────────────────
    Reddit      49 с     2 816     57.4 зап/с   229 MB

    Steam       50 с     999       20.0 зап/с   227 MB
    Хабр        136 с    188       1.4 зап/с    229 MB
    Лента.ру    73 с     88        1.2 зап/с    236 MB

Почему Хабр и Лента медленнее?

Двухэтапный парсинг: сначала список статей из поиска, затем полный текст
каждой статьи отдельным запросом. Скорость ограничена встроенным
rate limiter'ом для соблюдения этики сбора данных.

## Технические особенности

### Архитектура

Приложение построено на принципе Zero Infrastructure — не требует
установки баз данных, брокеров очередей или серверов. Все компоненты
работают локально.

    UI (Flet)
        └── AppController
                └── Dispatcher (multiprocessing spawn)
                        └── Worker Process
                                └── FallbackOrchestrator
                                        ├── LightExecutor  (curl_cffi, 15 MB RAM)
                                        └── StealthExecutor (Camoufox, ~200 MB RAM)
                                                └── JSONL файлы на диске

Импорты строго однонаправленные: core → engine → bots → ui.
Нарушение этого порядка является архитектурной ошибкой.

### Защита от блокировок

Finist использует трёхуровневую систему обхода защит:

1. TLS-маскировка
   Подмена fingerprint (JA3) под реальный браузер Chrome через curl_cffi.
   Потребление: 15 MB RAM. Обходит большинство статических блокировок.

2. Human Delay
   Случайные паузы между запросами (1–3 сек) через TokenBucket
   с адаптивным замедлением при получении HTTP 429.

3. Fallback к браузеру
   При обнаружении Cloudflare, капчи или кода 403 автоматически
   запускается Camoufox (Firefox с антидетект-патчами) с имитацией
   поведения человека: кривые Безье для мыши, рандомизированный скролл,
   ожидание авто-прохождения Cloudflare Turnstile.

### Потребление ресурсов

    Режим                   RAM         CPU
    ──────────────────────────────────────────
    Ожидание                ~110 MB     < 1%
    Парсинг (curl_cffi)     ~230 MB     5–15%
    Парсинг (Camoufox)      ~800-1400 MB     15–30%

### Хранение данных

- Данные пишутся построчно в JSONL (атомарная дозапись, нет потери при сбое)
- Каждая сессия — отдельная папка data/session_YYYY-MM-DD_HH-MM-SS/
- Экспорт в CSV и XLSX без использования pandas (только openpyxl)
- Логи ротируются автоматически (5 MB × 3 файла)

## Этика использования

Finist Crawler предназначен для исследовательских и образовательных целей.
При использовании:

- Соблюдайте robots.txt сайтов (rate limiter уже создан так, чтобы не нарушать правила сайтов — любые изменения в коде, которые могут повлечь бан ip, на совести пользователя)
- Не превышайте разумную частоту запросов (встроенный rate limiter помогает)
- Не собирайте персональные данные без необходимости
- Соблюдайте условия использования платформ
