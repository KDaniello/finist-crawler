## 1. Назначение инструмента

Finist Crawler создан для **исследовательских и образовательных целей**:

- Сбор корпусов текстов для лингвистического анализа
- Формирование учебных датасетов для занятий и курсов
- Исследование общественного дискурса в социальных медиа
- Мониторинг тональности отзывов для академических работ
- Журналистские расследования общественного мнения

Инструмент **не предназначен** для:

- Коммерческого ресейла собранных данных
- Создания спама или массовых рассылок
- Обхода платных подписок и закрытого контента
- Сбора персональных данных в целях слежки

## 2. Встроенные технические ограничения

Finist Crawler содержит несколько технических механизмов,
которые по умолчанию уже соблюдают базовую этику:

### Rate Limiter (TokenBucket)

Каждый источник имеет настроенный лимит запросов в секунду.
Значения подобраны так, чтобы не создавать нагрузку,
отличимую от активного пользователя-человека:

    Reddit      1.0 зап/с    (+ warmup паузы 0.7–2.1 сек)
    2GIS        1.5 зап/с
    Steam       0.9 зап/с
    Хабр        10.0 зап/с   (API позволяет)
    Лента.ру    3.0 зап/с
    Отзовик     1.5 зап/с    (браузер, медленнее по природе)

### Адаптивное замедление

При получении HTTP 429 (Too Many Requests) rate limiter
автоматически снижает скорость вдвое и продолжает снижать
при повторных 429. Максимальное замедление — 10x от базового.

### Human Delay

Между запросами добавляется случайная пауза (1–3 секунды)
с ±20% джиттером. Это имитирует поведение живого пользователя
и снижает риск автоматической блокировки.

### Предупреждение

Изменение параметров rate_limiter в YAML или коде Python
(увеличение requests_per_second, уменьшение задержек) может
привести к бану IP-адреса. Ответственность за такие изменения
лежит на пользователе.

## 3. Правила для пользователя

### Соблюдайте robots.txt

Перед сбором данных ознакомьтесь с файлом robots.txt сайта.
Он описывает, какие разделы запрещены для автоматического обхода.

Пример: https://www.reddit.com/robots.txt

Finist Crawler не проверяет robots.txt автоматически —
эта проверка остаётся на совести пользователя.

### Не собирайте персональные данные без необходимости

Данные пользователей (имена, аватары, ID) собираются только
в той мере, в которой они присутствуют в публичных API и необходимы
для идентификации записи. Не используйте собранные данные
для деанонимизации или профилирования конкретных людей.

### Соблюдайте Terms of Service платформ

Каждая платформа имеет пользовательское соглашение,
регулирующее автоматический сбор данных:

    Reddit   https://www.redditinc.com/policies/data-api-terms
    Steam    https://store.steampowered.com/privacy_agreement/
    2GIS     https://law.2gis.ru/api-rules

Убедитесь, что ваш сценарий использования не нарушает эти правила.

### Не перегружайте серверы

Используйте минимально необходимое количество страниц.
Если вам нужно 100 отзывов — не устанавливайте лимит 10 000.

## 4. Правовой контекст

### В России

Сбор публично доступной информации из интернета
в исследовательских и образовательных целях
в целом не противоречит российскому законодательству,
при условии соблюдения следующих требований:

- Данные получены из публично доступных источников
- Данные не являются персональными по смыслу 152-ФЗ
  (или обезличены перед анализом)
- Собранные данные не передаются третьим лицам
  в коммерческих целях

Нормы 273-ФЗ (о компьютерной информации) не применяются
к получению общедоступных данных через стандартные HTTP-запросы.

### В ЕС / США

В академическом и журналистском контексте сбор публичных данных
защищён принципом Fair Use (США) и исследовательским исключением
в GDPR (ЕС), при условии минимизации данных и отсутствия
коммерческого использования.

Это не юридическая консультация.
При сомнениях обратитесь к специалисту по информационному праву.

## 5. Что делать при сомнениях

Если вы не уверены, допустимо ли собирать данные с конкретного сайта:

1. Прочитайте robots.txt и Terms of Service сайта
2. Проверьте, есть ли у сайта официальный API
   (например, Reddit и Steam предоставляют официальные API)
3. Напишите в поддержку сайта с описанием вашего исследования
4. Используйте минимально необходимый объём данных
5. Обезличьте данные перед публикацией результатов исследования
