## 1. Общие поля

Каждая запись, независимо от источника, содержит три обязательных поля.

    external_id   Уникальный идентификатор записи.
                  Либо берётся из API источника, либо вычисляется
                  детерминированно через MD5 от (url + автор + текст).
                  Формат для вычисленных ID: "finist-<16 hex символов>"

    message_url   Прямая ссылка на страницу, с которой взята запись.

    metadata      Служебный объект.
                  metadata.extracted_at — ISO 8601 timestamp момента сбора.

## 2. Reddit

Источник: reddit_comments.yaml
Стратегия: двухэтапная (поиск постов → рекурсивное извлечение комментариев)

    {
      "author":      "имя пользователя Reddit",
      "text":        "текст комментария (переносы строк заменены пробелом)",
      "likes":       "суммарный рейтинг (score)",
      "created_at":  "Unix timestamp создания",
      "depth":       "глубина вложенности (0 = верхний уровень)",
      "title":       "заголовок родительского поста",
      "reply_to_id": "ID родительского комментария (t1_xxxxx)",
      "external_id": "ID комментария из Reddit API",
      "message_url": "https://www.reddit.com/r/.../comments/.../.json",
      "metadata":    {"extracted_at": "2024-01-01T00:00:00+00:00"}
    }

Особенности:
- Комментарии извлекаются рекурсивно, включая вложенные ответы
- Удалённые комментарии ([deleted], [removed]) пропускаются
- Поле depth позволяет восстановить дерево обсуждения
- Скорость ограничена до 1 рек/с из-за политики Reddit API

## 3. 2GIS

Источник: twogis_search.yaml
Стратегия: двухэтапная (HTML поиск → JSON API отзывов)

    {
      "external_id": "finist-<16 hex>  или  ID из API 2GIS",
      "text":        "текст отзыва",
      "author":      "имя пользователя",
      "created_at":  "ISO 8601 дата создания (2024-01-15T10:30:00)",
      "rating":      "оценка от 1 до 5",
      "message_url": "https://public-api.reviews.2gis.com/...",
      "metadata":    {"extracted_at": "2024-01-01T00:00:00+00:00"}
    }

Особенности:
- Первый этап: парсинг HTML страницы поиска 2GIS, извлечение ID организаций
  через regex из ссылок вида /moscow/firm/4504127908559765
- Второй этап: запросы к открытому JSON API отзывов без авторизации
- Пагинация через offset (50 отзывов за запрос)
- Самый быстрый источник: до 52 записей/сек

## 4. Steam

Источник: steam_reviews.yaml
Стратегия: одноэтапная (прямой API отзывов по App ID)

    {
      "review_id":                "ID отзыва в Steam",
      "author_steamid":           "64-битный Steam ID автора",
      "playtime_at_review_hours": 12.5,
      "playtime_total_hours":     87.3,
      "is_positive":              "True или False",
      "text":                     "текст отзыва",
      "votes_up":                 "количество лайков",
      "votes_funny":              "количество отметок «смешно»",
      "created_at":               "Unix timestamp создания",
      "language":                 "язык отзыва (russian, english...)",
      "steam_purchase":           "True — куплено в Steam",
      "game_appid":               "числовой App ID игры",
      "message_url":              "https://store.steampowered.com/appreviews/...",
      "metadata":                 {"extracted_at": "2024-01-01T00:00:00+00:00"}
    }

Особенности:
- playtime_at_review_hours и playtime_total_hours конвертированы из минут в часы
- Пагинация через cursor (Steam API): значение cursor меняется с каждой страницей
- Конец данных: cursor не изменился или reviews пустой
- Поле language позволяет фильтровать отзывы по языку

## 5. Хабр

Источник: habr_search.yaml
Стратегия: двухэтапная (Search API → API статьи)

    {
      "external_id": "числовой ID статьи на Хабре",
      "title":       "заголовок статьи (HTML-теги включены)",
      "author":      "алиас автора",
      "created_at":  "ISO 8601 дата публикации",
      "views":       "количество просмотров",
      "likes":       "рейтинг (score)",
      "comments":    "количество комментариев",
      "text":        "полный текст статьи (HTML)",
      "message_url": "https://habr.com/kek/v2/articles/.../",
      "metadata":    {"extracted_at": "2024-01-01T00:00:00+00:00"}
    }

Особенности:
- Внутренний API Хабра (/kek/v2/) возвращает данные в JSON
- Поле text содержит HTML-разметку статьи
- Пагинация списка через offset (20 статей за страницу)
- Медленнее других API-источников из-за двух последовательных запросов

## 6. Лента.ру

Источник: lenta_search.yaml
Стратегия: двухэтапная (Search API v2 → HTML статьи)

    {
      "external_id":  "числовой docid из Search API",
      "title":        "заголовок статьи",
      "excerpt":      "краткое описание из поиска",
      "text":         "полный текст статьи",
      "author":       "имя автора (из json-topic-info или CSS)",
      "description":  "дополнительное описание (из json-topic-info)",
      "alt_headline": "альтернативный заголовок",
      "created_at":   "ISO 8601 дата публикации",
      "type":         "Новость / Статья / Мнение / ...",
      "rubric":       "Россия / Мир / Экономика / ...",
      "image_url":    "URL главного изображения",
      "source":       "lenta.ru",
      "message_url":  "https://lenta.ru/articles/...",
      "metadata":     {"extracted_at": "2024-01-01T00:00:00+00:00"}
    }

Особенности:
- Фильтрация типов: галереи (3), видео (6), онлайны (8) пропускаются,
  собираются только текстовые материалы (тип 1, 2, 4, 5)
- Полный текст извлекается приоритетно из <script class="json-topic-info">
  (JSON, вшитый Лентой прямо в страницу), при неудаче — через CSS-селекторы
- Пагинация Search API через параметр from= (offset)
- API возвращает максимум 10 000 результатов

## 7. Отзовик

Источник: otzovik_reviews.yaml
Стратегия: двухэтапная (HTML список → HTML страница отзыва)
Метод: браузерный (Camoufox)

    {
      "title":       "заголовок отзыва",
      "text":        "полный текст отзыва",
      "author":      "имя пользователя",
      "rating":      "оценка (1-10)",
      "created_at":  "дата публикации (YYYY-MM-DD)",
      "likes":       "количество лайков",
      "comments":    "количество комментариев",
      "message_url": "https://otzovik.com/review_XXXXX.html",
      "metadata":    {"extracted_at": "2024-01-01T00:00:00+00:00"}
    }

Особенности:
- Требует браузера (Camoufox): Яндекс SmartCaptcha активируется при curl
- При появлении капчи показывается таймер 300 секунд в UI мониторинга
- Пользователь должен решить капчу в открытом окне браузера вручную
- Скорость ~0.3 записи/сек из-за рендеринга страниц
- Поле direct_url передаётся напрямую (не через keyword)

## 8. Форматы экспорта

### CSV

- Кодировка: UTF-8 с BOM (utf-8-sig) для корректного открытия в Excel
- Разделитель: точка с запятой (;) — совместимо с русской локалью Windows
- Переносы строк внутри ячеек заменяются пробелами
- Объекты (dict, list) сериализуются в JSON-строку

### XLSX

- Движок: openpyxl (без pandas)
- Заголовки: жирный белый шрифт, синий фон (#4472C4)
- Текстовые поля (text, body, description): перенос по словам включён
- Ширина колонок: автоподбор, максимум 80 символов
- Первая строка заморожена (freeze_panes="A2")
- Автофильтр включён на все колонки

## 9. Хранение на диске

    data/
    └── session_YYYY-MM-DD_HH-MM-SS-ffffff/
        └── <source_key>/
            ├── <source_key>.jsonl
            └── <source_key>_export.csv  (или .xlsx, после экспорта)

JSONL (JSON Lines) — каждая строка является самостоятельным JSON объектом.

Преимущества над SQLite/CSV как основного формата хранения:
- Нет необходимости загружать весь файл в память
- Дозапись атомарна: сбой на строке N не портит строки 1..N-1
- Человекочитаем в любом текстовом редакторе
- Прямая совместимость с pandas, polars, DuckDB для дальнейшего анализа
